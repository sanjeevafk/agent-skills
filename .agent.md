# agent-skills — Contributor Guide

**Purpose:** Global repository of reusable Claude Code skills, commands, agents, and MCP servers  
**Structure:** Organized by skill domain + shared infrastructure  
**Maintenance:** Curated library with Tirith security scanning before all releases

## Key Directories

- `skills/` — Individual skill folders (400+), each with SKILL.md + resources
- `commands/` — Slash commands organized by category
- `agents/` — Autonomous agents for multi-step tasks
- `mcp-servers/` — Model Context Protocol server integrations
- `.github/` — Workflows, issue templates, documentation
- `docs/` — Comprehensive skill usage guides
- `security/` — Tirith scanning configuration and best practices

## Important Files

- `README.md` — Overview and quick start
- `DISTRIBUTION.md` — How skills are discovered and installed
- `SKILL_TEMPLATE.md` — Template for creating new skills
- `CONTRIBUTING.md` — Guidelines for skill contributions
- `engineering-principles.md` — Workspace governance and response style

## When Adding a New Skill

1. **Create folder:** `skills/my-skill/`
2. **Add SKILL.md:** Include when-to-use, description, examples
3. **Add resources:** Code, templates, data files as needed
4. **Test locally:** Verify skill triggers and functionality
5. **Run security scan:** `npx @agent-skills/tirith scan skills/my-skill/`
6. **Create PR:** Include test results and use cases

## Skill Quality Standards

- **Discovery:** When-to-use description must clearly signal relevance
- **Safety:** All subagent calls and file operations auditable
- **Token Efficiency:** Lightweight SKILL.md, heavy logic in runnable code
- **Documentation:** Examples, use cases, and edge cases covered
- **Testing:** All skills tested before merging to main
- **Response Style:** No emojis in responses; use clear text and formatting

## Common Patterns

**Parallel Agents:** Use `runSubagent` with multiple agents simultaneously  
**File Operations:** Use `read_file`, `grep_search`, `semantic_search` in parallel batches  
**Long Tasks:** Break into smaller tracked steps, use `manage_todo_list`  
**Security:** Always scan skills with Tirith before contributing to this library

## Before Submitting a PR

1. Review `CONTRIBUTING.md` for submission guidelines
2. Check that SKILL.md when-to-use description is precise
3. Run Tirith security scan: `npx @agent-skills/tirith scan skills/your-skill/`
4. Verify all examples in SKILL.md match actual skill behavior
5. Include test results and use case validation

## Token Optimization

This repo prioritizes lean skill descriptors to avoid token waste:
- SKILL.md kept concise (discovery focus only)
- Heavy logic lives in referenced code, not prompts
- Skills load on-demand, not globally
- Engineering principles available in `engineering-principles.md`
